Arquitetura Plugin

Cada módulo implementa BaseModule e pode ser executado isoladamente ou integrado à suíte. Lazy-loading garante startup rápido.

01

FLOWSHEET

Diagrama de fluxo de processo (PFD) com balanço de massa automático via algoritmo de Kahn. Suporte a 50+ equipamentos industriais, ~300 componentes químicos e 12 portas de conexão por nó.

BPMN 2.0

Modelador de processos conforme especificação OMG BPMN 2.0. Swimlanes, eventos, gateways exclusivos, paralelos e inclusivos, tarefas de usuário, serviço e subprocessos com auto-layout.

OMG BPMN 2.0SwimlanesGatewaysAuto-layout

WBSPMBOKHierárquicoAuto-WBS

Filosofia
de Projeto

Design Industrial Criativo

Design que auxilia na clareza do pensamento e na detecção de falhas — não apenas na estética da interface.

// 01

Neo-Brutalismo Industrial

Bordas sólidas 4px, sombras hard offset 8×8px, tipografia pesada uppercase. Um vocabulário visual que prioriza hierarquia e legibilidade — ideal para ambientes onde clareza é segurança operacional.

// 02

Arquitetura Modular

Plugin architecture: o core gerencia apenas navegação e estado global. Cada módulo é completamente independente — pode rodar isolado com python -m proeng.modules.X ou em conjunto via suíte completa.

// 03

Software Livre GPLv3

Distribuído sob GNU GPLv3. A copyleft forte garante que o ProEng permaneça livre para sempre — liberdade de usar, estudar, modificar e redistribuir com as mesmas liberdades preservadas.

// 04

Persistência Total

Estado completo de todos os módulos serializado em JSON no formato .proeng. Cada diagrama, corrente e card preservados com fidelidade absoluta via get_state() / set_state().

// 05

14 Temas Dinâmicos

Troca de tema em tempo real via refresh_theme_recursive() — propaga instantaneamente para todos os módulos, diagramas, toolbars e diálogos. Design neo-brutalista consistente em todos os 14 esquemas.

// 06

Export Profissional

PNG com 3× supersampling para qualidade de publicação científica. PDF A3 landscape para relatórios técnicos formais. Exportação individual via _export_view(view, fmt) em qualquer módulo.

Lazy Loading

Módulos instanciados somente no primeiro acesso. Startup rápido mesmo com 11 módulos pesados — sem custo de memória antecipado.

T() Singleton

Acesso ao tema ativo via T()["accent"]. refresh_theme_recursive(w) propaga para todos os widgets filhos automaticamente.

Persistência JSON

Formato .proeng: snapshot completo. Cada módulo implementa get_state() e set_state() conforme a interface BaseModule.

Vector Rendering

QGraphicsScene com aceleração por hardware. Polígonos, gradientes dinâmicos e sombras renderizados em tempo real com pixel-perfect precision.

BaseModule

Contrato único: get_state, set_state, refresh_theme, get_view. Garante integração perfeita de qualquer módulo externo.

Export Engine

PNG 3× supersampling via _export_view(view,"png"). PDF A3 landscape via _export_view(view,"pdf"). Qualidade de publicação técnica.

PASSOS

Instalação direta via requirements.txt, cobrindo interface, exportação Excel e stack numérica do projeto. Funciona em Windows 10/11, Linux (Ubuntu/Debian) e macOS.

Python 3.8+ requirements.txt PyQt5 + openpyxl Windows · Linux · macOS 1366×768 min.

01
Clonar repositório
git clone https://github.com/sasal0519/proeng.git cd proeng
02
Criar ambiente virtual
python -m venv .venv # Windows: .venv\Scripts\activate # Linux/Mac: source .venv/bin/activate
03
Instalar dependências
pip install -r requirements.txt
04

python main.py # Ou módulo individual: python -m proeng.modules.flowsheet
